# 🏥 Health Log Pro (健康日志)

> 一个极简、隐私优先的个人健康全流程记录工具。
> A minimalist, privacy-first personal health tracking PWA.

## 📖 项目简介

**Health Log Pro** 旨在帮助用户以线性“病程”的视角记录健康状况。不同于传统的打卡软件，它允许用户创建特定的生病周期（如“甲流”、“过敏”），并将症状、用药、就医记录关联到该周期中，从而生成清晰的病情发展时间轴。

应用采用 **PWA (Progressive Web App)** 技术，可直接安装到手机桌面，体验接近原生 App，且数据完全存储在本地，保障隐私安全。

## ✨ 核心功能 (V11.0)

### 1. 🤒 全流程病程管理 (Illness Courses)
* **多线程记录**：支持同时记录多个进行中的病程（如同时记录“慢性胃炎”和“突发感冒”）。
* **智能时间轴**：自动计算病程持续天数（Day 1, Day 2...），以时间轴形式直观展示病情演变。
* **就医档案**：支持记录就诊日期、科室、医生诊断及处方医嘱。
* **归档机制**：病程结束后可标记“已康复”，归档至历史记录。

### 2. 📝 多维健康记录
* **身体不适**：
    * 可视化选择部位（头、胸、腹等）。
    * 1-10 级疼痛/严重程度滑块。
    * 支持标记“病情演变/转折点”（高亮显示）。
    * 支持补录日期（自动处理时序）。
* **用药治疗**：
    * 记录药品名称、剂量、使用原因。
    * 支持多种给药方式（口服、外用、注射、吸入等）。

### 3. 🎨 极致 UI/UX 设计
* **灵动岛式导航**：底部悬浮胶囊导航（概览/统计/历史），配合半透明毛玻璃效果 (`Backdrop Blur`)。
* **竖向堆叠菜单**：右下角 FAB 按钮点击后，以竖向动画弹出操作菜单，单手操作更友好。
* **沉浸式体验**：完美适配 iOS 刘海屏/灵动岛 (`safe-area-inset`)，全屏无缝融合。
* **极简视觉**：去除了冗余的时间显示，信息展示更加高效。

### 4. 📊 数据统计与回溯
* **全局搜索**：支持按症状、药名、病程名、日期、备注内容进行全量搜索。
* **健康月历**：热力图风格展示每月的健康状况分布。
* **部位分析**：自动统计高频不适部位。

### 5. 🛡️ 数据安全
* **本地存储**：所有数据存储在浏览器 `LocalStorage` 中，不上传任何服务器。
* **数据备份**：支持一键导出 JSON 备份文件，支持从 JSON 文件恢复数据。

## 🚀 快速开始

### 本地运行

1.  克隆仓库：
    ```bash
    git clone [https://github.com/your-username/my-health-log.git](https://github.com/your-username/my-health-log.git)
    cd my-health-log
    ```

2.  安装依赖：
    ```bash
    npm install
    ```

3.  启动开发服务器：
    ```bash
    npm run dev
    ```
    访问 `http://localhost:5173` 即可预览。

### 部署上线

本项目对 **Vercel** 零配置支持：

1.  将代码推送到 GitHub。
2.  在 Vercel 中导入该仓库。
3.  点击 **Deploy**，即可获得在线地址。

## 📱 如何安装到手机

本应用支持 PWA，无需通过 App Store 即可安装。

### iOS (iPhone/iPad)
1.  使用 **Safari** 浏览器打开网站。
2.  点击底部中间的 **分享按钮** (方框带箭头)。
3.  下滑找到并点击 **“添加到主屏幕” (Add to Home Screen)**。
4.  回到桌面，点击图标即可全屏运行。

### Android
1.  使用 **Chrome** 打开网站。
2.  点击右上角菜单 -> **“安装应用”** 或 **“添加到主屏幕”**。

## ⚠️ 数据安全提示

* **数据位置**：数据保存在您手机浏览器的本地缓存中。
* **风险**：**删除桌面图标（App）**或**清除浏览器缓存**会导致数据**永久丢失**。
* **建议**：请务必定期在“设置”页面点击 **“导出备份”**，将 JSON 文件保存到安全的地方（如 iCloud Drive 或 微信文件传输助手）。
